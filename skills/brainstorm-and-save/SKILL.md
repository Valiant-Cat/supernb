# Brainstorm And Save

Use this mode for idea exploration that must result in local documents, not just chat output.

## Workflow

1. Clarify the idea and objective.
2. Explore alternatives and tradeoffs.
3. Save the result into the initiative artifacts if they exist.
4. If no initiative scaffold exists, create one or save into a clearly named local markdown document.

## Output Standard

The saved output should include:

- the problem being solved
- target user or audience
- possible approaches
- recommended direction
- open questions

## Escalation Rule

If the brainstorming outcome is likely to become a product build, hand off to `product-research-prd` or `full-product-delivery`.
